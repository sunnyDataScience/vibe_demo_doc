## 🚀 一鍵生成資料說明文件

### 基本指令
將以下指令複製貼上，替換 `[檔案路徑]` 和 `[資料來源]`：

```
請使用以下系統 prompt 分析我的資料集：

你是一位專業的資料分析師和技術文件撰寫專家。請為提供的資料集創建一份詳細、結構化的中英文資料說明文件。

**分析目標：**
- 資料檔案：[檔案路徑]
- 資料來源：[資料來源網址或描述]

**請按照以下9大結構生成完整文件：**
1. 標題與概述（含基本資訊、時間範圍、地理涵蓋）
2. 資料集描述（業務背景、收集目的、記錄意義）
3. 欄位說明（分組表格：欄位名稱、資料型態、詳細說明）
4. 資料特徵（關鍵指標、分布特性、限制）
5. 適用分析類型（描述性、客戶、財務、預測四大類）
6. 資料品質注意事項（完整性、偏差、使用建議）
7. 使用建議（初學者、中階、進階分層）
8. 程式碼範例（Python載入、探索性分析）
9. 相關資源（來源連結、授權資訊）

請使用 Markdown 格式，中文為主，重要術語提供英文對照。
```

## 🔧 進階客製化

### 針對特定分析需求

**銷售分析重點：**
```
特別強調以下分析方向：
- 銷售趨勢和季節性模式
- 產品績效和盈利分析
- 客戶貢獻度分析
- 價格彈性和促銷效果
```

**客戶分析重點：**
```
特別強調以下分析方向：
- 客戶分群和行為模式

- 客戶生命週期價值

**財務分析重點：**
```
特別強調以下分析方向：
- 營收和成本結構分析
- 毛利和淨利分析
- 現金流預測
- 財務比率分析

## 🛠️ 故障排除

### 常見問題

**Q: 生成的文件太簡單？**
A: 在 prompt 中加入「請提供更詳細的分析建議和具體的實作步驟」

**Q: 缺少特定領域的分析？**
A: 明確指出領域需求，如「這是電商資料，請特別關注轉換率和客戶獲取成本分析」

**Q: 程式碼範例不夠實用？**
A: 要求「請提供完整的端到端分析範例，包含資料清理到可視化」

**Q: 欄位說明不夠詳細？**
A: 補充說明「請深入分析每個欄位的業務意義和潛在的分析價值」

### 改進建議模板

```
請改進剛才的分析文件：

【需要加強的部分】
- 欄位說明：請加入更多業務解釋
- 分析建議：請提供更具體的實作步驟
- 程式碼範例：請加入資料清理和視覺化
- 使用情境：請針對 [具體業務場景] 提供建議

【額外需求】
- [任何特殊要求]
```
