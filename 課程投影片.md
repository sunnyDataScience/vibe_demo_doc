這是一份經過完整重構、針對「商管小白大學生」設計的 **12 小時 AI Vibe Coding 工作坊架構**。

本設計結合了 **《Ai Vibe Coding 12h Workshop V1》** 的核心精神（雙鑽石模型、產品思維），以及 **《投影片製作系統化指引教學》** 中的教學設計原則（蓋聶九大學習事件、SOIL 模式）。

特別針對您的需求，在 **M1** 加入了 **SDD (軟體設計文件) 與 Generative UI**，並在 **M2** 建立了 **Coding Agent 的深度評測與選型矩陣**。

### 🟢 第一部分：整體設計策略與視覺規範 (Vibe & UI/UX)

為了讓商管學生進入「Vibe Coding」狀態，投影片風格需跳脫傳統「程式課」的生硬感，轉向「現代產品經理工作坊」的風格。

#### 1. 教學設計原則 (Pedagogy)

* **蓋聶九大學習事件 (Gagne’s 9 Events)：** 每個模組均包含「引起注意 $\rightarrow$ 呈現內容 $\rightarrow$ 引導練習 $\rightarrow$ 提供回饋」的循環。
* **認知負荷管理 (SOIL 模式)：** 程式碼與技術名詞將被「區塊化 (Chunking)」處理，避免資訊過載。
* **行動導向標題 (Action Title)：** 每頁標題直接說明該頁的結論或行動，而非僅是名詞（例如：用「SDD 是給 AI 看的施工圖」取代「SDD 介紹」）。

#### 2. 視覺規範 (Visual Identity)

* **字體：** 標題使用 **微軟正黑體 (Bold)** 或 **思源黑體 (Heavy)** (32-40pt)；內文使用無襯線體 (24-28pt)。
* **配色：**
  * **背景：** 深色模式 (Dark Mode - 深藍或炭灰) 以營造 Coding Vibe 專業感。
  * **強調色：** 霓虹綠或亮黃色 (用於 Highlight 程式碼與關鍵 Prompt)。
* **版面：** 遵循 **CRAP 原則** (對比、重複、對齊、親密性)，多用左右分割佈局 (左概念/右操作)。

### 🟢 第三部分：實作環節規劃與喘息空間

為了避免「填鴨式教學」，本課程嚴格遵循 **40% 講解 + 60% 實作** 的比例，並預留「緩衝區」。

#### 1. 喘息空間 (Buffers)

* **Debug Buffer (M3/M4):** 這是新手最容易卡關的地方。預留 20 分鐘「彈性時間」。
  * *若進度超前：* 進行 UI 優化挑戰 (加個 Dark Mode)。
  * *若進度落後：* 講師直接提供 `solution.py`，讓學生跟上，避免挫折感。
* **隱性休息：** 利用「小組互評」、「Prompt 迭代分享」作為腦力轉換的休息。

#### 2. 關鍵實作清單 (Hands-on Labs)

1. **Lab 0 (M0):** 痛點盤點畫布 — 找出你最想自動化的事。
2. **Lab 1 (M1):** PRD Lite + SDD (Mermaid) + Generative UI — 設計你的財務工具規格與介面。
3. **Lab 2 (M2):** 安裝 Cursor 並執行 Hello World — 準備你的 Vibe 武器。
4. **Lab 3 (M3):** 先救自己 — 整合散落的帳單/交易紀錄/Excel。
5. **Lab 4 (M4):** 讓數據更聰明 — 串匯率/股價/公開資料 API。
6. **Lab 5 (M5):** 把它交出去 — 部署到 Streamlit Cloud，產生連結。
