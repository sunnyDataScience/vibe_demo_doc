## 基礎操作類 📂

### 讀取數據
```
"請讀取 [檔案名稱.csv]，
編碼為 [utf-8/big5/gbk]，
顯示：
1. 前5筆資料
2. 資料形狀（筆數、欄位數）
3. 欄位資訊（名稱、類型、非空值數）
4. 基本統計描述"
```

### 數據概覽
```
"請分析這份數據：
1. 總共有多少筆記錄和欄位
2. 各欄位的資料類型
3. 缺失值數量和比例
4. 數值欄位的分布情況
5. 類別欄位的唯一值數量"
```

### 數據篩選
```
"請篩選出符合以下條件的資料：
- [欄位A] [大於/小於/等於] [值]
- [欄位B] 在 [值1, 值2, 值3] 中
- [欄位C] 包含 [文字]
顯示筆數和前10筆"
```

### 數據分組
```
"請按 [欄位名] 分組，
計算：
- 每組的筆數
- [數值欄位1] 的總和
- [數值欄位2] 的平均值
- [數值欄位3] 的最大/最小值
按 [排序欄位] 降序排列"
```

### 數據合併
```
"請合併兩個資料表：
表1：[表名或變數名]
表2：[表名或變數名]
合併方式：[inner/left/right/outer]
關聯欄位：[欄位名]
處理重複欄位名：加後綴 _1 和 _2"
```

## 數據清理類 🧹

### 品質檢查
```
"請執行數據品質檢查：
1. 缺失值分析
   - 各欄位缺失數量和百分比
   - 缺失模式（隨機/系統性）
2. 重複值檢測
   - 完全重複的記錄數
   - 基於 [關鍵欄位] 的重複
3. 異常值識別
   - 使用IQR方法
   - 標記超出合理範圍的值
4. 一致性檢查
   - 日期格式
   - 類別值拼寫
產生品質評分報告"
```

### 缺失值處理
```
"[欄位名] 有 [X]% 的缺失值，
資料類型是 [數值/類別/日期]，
請使用以下策略處理：
- 數值型：[均值/中位數/眾數/0] 填補
- 類別型：[眾數/'未知'/特定值] 填補
- 日期型：[前向/後向] 填補
並說明選擇此策略的理由"
```

### 異常值處理
```
"請檢測並處理 [欄位名] 的異常值：
1. 使用 [IQR/3倍標準差/業務規則] 方法檢測
2. 顯示異常值數量和範例
3. 處理方式：[刪除/截尾/標記/保留]
4. 顯示處理前後的分布對比"
```

### 資料轉換
```
"請進行以下資料轉換：
1. 日期處理
   - 將 [欄位] 轉為日期格式
   - 提取 [年/月/週/星期幾]
2. 類別編碼
   - 對 [欄位] 進行 [one-hot/標籤] 編碼
3. 數值轉換
   - 對 [欄位] 進行 [標準化/正規化/對數] 轉換
4. 新增欄位
   - 根據 [條件] 創建 [新欄位名]"
```

## 分析執行類 📊

### 相關性分析
```
"請執行相關性分析：
1. 計算所有數值欄位的相關係數矩陣
2. 識別高度相關的變數對（|r| > 0.7）
3. 用熱力圖視覺化相關矩陣
4. 解釋關鍵發現的業務含義"
```

### 趨勢分析
```
"請分析 [時間欄位] 對 [目標變數] 的趨勢：
1. 繪製時間序列圖
2. 計算移動平均（[7/30/90]天）
3. 識別趨勢（上升/下降/平穩）
4. 標注異常點和轉折點
5. 分解季節性成分（如果有）"
```

### 分布分析
```
"請分析 [欄位名] 的分布：
1. 繪製直方圖和密度圖
2. 計算統計量（均值、中位數、偏度、峰度）
3. 進行正態性檢驗
4. 識別分布類型（正態/偏態/雙峰等）
5. 提供業務解釋"
```

### 比較分析
```
"請比較不同 [分組變數] 的 [目標變數]：
1. 計算各組的描述統計
2. 進行統計檢驗（t-test/ANOVA）
3. 繪製箱形圖或小提琴圖
4. 識別顯著差異
5. 提供行動建議"
```

### RFM分析
```
"請對客戶交易數據執行RFM分析：
1. 計算每個客戶的：
   - Recency：最近一次購買距今天數
   - Frequency：購買次數
   - Monetary：總消費金額
2. 使用五分位數進行評分（1-5分）
3. 組合RFM分數定義客戶群：
   - Champions (555, 554, 544)
   - Loyal Customers (543, 444, 435)
   - At Risk (255, 254, 245)
   - Lost (111, 121, 131)
4. 計算每群的：
   - 客戶數和佔比
   - 平均RFM值
   - 收入貢獻
5. 為每個群體提供行銷策略"
```

### 同期群分析
```
"請執行用戶留存的同期群分析：
1. 按 [註冊月份/首購月份] 定義同期群
2. 計算每個同期群在第 [1,2,3,6,12] 個月的留存率
3. 創建留存矩陣和熱力圖
4. 識別留存模式和問題群組
5. 提供改善建議"
```

## 視覺化類 📈

```

- 加入：[網格/數值標籤/註解]"
```

### 多子圖布局
```
"請創建 [2×2/2×3/3×3] 的子圖布局：
子圖1（左上）：[圖表類型] - [描述]

共用配色，加入子圖標題"
```

### 互動式圖表
```
"請用 plotly 創建互動式 [圖表類型]：
- 基本設定同上
- 滑鼠懸停顯示：[詳細資訊列表]
```

## 報告產出類 📑

### 分析總結
```
"根據以上分析結果，請總結：
1. 關鍵發現（3-5個要點）
   - 用數據支撐每個發現
   - 量化影響程度
2. 業務洞察
   - 解釋發現的商業含義
   - 識別機會和風險
3. 對比分析
   - 與 [歷史/目標/競品] 對比
   - 標注顯著變化"
```

### 行動建議
```
"基於分析發現，請提供建議：
1. 立即行動（Quick Wins）
   - [具體行動1]
   - 預期效果：[量化指標]
   - 所需資源：[人力/時間/預算]
2. 短期計劃（1-3個月）
   - [具體行動2]
   - 實施步驟：[步驟列表]
   - 成功指標：[KPI]
3. 長期策略（3-12個月）
   - [策略方向]
   - 里程碑：[時間點和目標]
   - 風險評估：[潛在風險和對策]"
```

### 執行摘要
```
"請為這份分析撰寫執行摘要：
背景：[1-2句話說明為什麼做這個分析]
方法：[1句話說明分析方法]
關鍵發現：
• [發現1]：[一句話+關鍵數字]
• [發現2]：[一句話+關鍵數字]
• [發現3]：[一句話+關鍵數字]
建議：[最重要的1-2個行動建議]
預期影響：[量化的業務影響]
限制200字以內"
```

### 技術文檔
```
"請為這個分析專案生成技術文檔：
1. 專案概述
   - 目的和範圍
   - 關鍵假設
2. 數據說明
   - 數據來源和時間範圍
   - 欄位定義（資料字典）
   - 數據品質說明
3. 方法論
   - 分析流程圖
   - 使用的統計方法
   - 參數設定和理由
4. 程式碼說明
   - 主要函數功能
   - 關鍵邏輯解釋
   - 相依套件清單
5. 結果解釋
   - 輸出格式說明
   - 指標計算公式
   - 限制和注意事項"
```

## 使用技巧 💡

### Prompt 優化原則
```
1. 明確具體
   ❌ "分析數據"
   ✅ "分析sales.csv中的月銷售趨勢"

2. 結構清晰
   使用編號、分點、階層結構

3. 提供背景
   說明數據來源、業務場景、預期用途

4. 指定輸出
   明確要程式碼、解釋、或圖表

5. 迭代改進
   根據回應調整和細化要求
```

### 進階技巧
```
1. 角色設定
   "你是一位資深數據分析師..."

2. 少樣本學習
   "像這個例子一樣：[範例]"

3. 思維鏈
   "請一步一步思考並解釋..."

4. 溫度調節
   創意任務：較高溫度
   精確任務：較低溫度
```

### 常見場景對應
| 場景 | 建議模板 | 關鍵詞 |
|:-----|:---------|:-------|
| 探索數據 | 數據概覽 | info, describe, head |
| 找問題 | 品質檢查 | 缺失值, 異常值, 重複 |
| 做分析 | 分析執行類 | 相關, 趨勢, 分布 |
| 展示結果 | 視覺化類 | 圖表, 儀表板 |
| 寫報告 | 報告產出類 | 總結, 洞察, 建議 |
