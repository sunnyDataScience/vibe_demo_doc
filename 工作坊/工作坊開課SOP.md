# AI Vibe Coding 工作坊開課 SOP

> 2 天 / 12 小時 | Case Study + 實作導向 | 目標：課堂內完成成功經驗，回家能立刻複製

## A) 一句話 Insight

**「讓學員帶走一個能用的連結，而非一堆聽過的概念。」**

## C) 課程節奏模板

每輪 90-120 分鐘，固定結構：

| 階段 | 時間 | 目的 | 產出 | DoD | Fallback |
|:-----|:-----|:-----|:-----|:----|:---------|
| **Context** | 15min | 建立心智模型、說明「為什麼要做」 | 理解任務目標 | 能用一句話說出本輪要做什麼 | 放慢速度，用類比重講 |
| **Demo** | 15min | 展示完整流程，降低未知恐懼 | 看過一遍成功案例 | 知道終點長什麼樣 | 切換成 GIF/錄影 |
| **Hands-on** | 45-60min | 動手做，遇到卡關就學 | 可運作的半成品 | 程式能跑 or UI 能看 | 5 分鐘修不好 → 發 solution.py |
| **Reflection** | 10min | 小組互評、分享卡關點 | 3 個 Key Takeaways | 每組說出一個學到的事 | 講師直接總結 |
| **休息** | 10min | 腦力恢復 | — | — | — |

## D) 專案設計：三段梯度

| 階段 | 模組 | 目標成功率 | 產出型態 | 評估方式 |
|:-----|:-----|:----------|:---------|:---------|
| **暖身** | M0-M2 | 95% | PRD Lite + SDD 架構圖 + Hello World | 能貼上 Notion、能跑出結果 |
| **核心** | M3-M4 | 80% | 整合報表 + API 串接成功 | 本地執行無錯誤 |
| **挑戰** | M5 | 70% | 雲端部署連結 + 3min Demo | 連結可開、能說出效益 |

**成功指標**：學員帶走一個「可分享的連結」+「能說出省了多少時間」

## F) 角色分工 R&R

| 角色 | 職責 | 關鍵時刻 |
|:-----|:-----|:---------|
| **講師** | 講解 Context/Demo、控制節奏、處理共通問題 | 開場/收尾/Demo Day 主持 |
| **助教 A** | 負責左半場學員、即時救火、發 solution.py | Hands-on 期間巡場 |
| **助教 B** | 負責右半場學員、管簽到表、拍照記錄 | 開場簽到 / 收尾問卷 |

### 助教救援決策樹

```
學員舉手求救
    │
    ├─ 能在 2 分鐘內解決？ → 直接處理
    │
    └─ 不行？
        │
        ├─ 是個人問題（環境/帳號）？ → 先給 solution.py 讓他跟上，課後再處理
        │
        └─ 是共通問題（>3 人卡同處）？ → 通知講師暫停，全班一起處理
```

**5 分鐘修不好規則**：
1. 標記學員名字 + 卡關點
2. 發放對應 Checkpoint 的 solution.py
3. 確認學員能跑起來後繼續
4. 課後統一追蹤處理

## H) KPI 與檢核

| 指標 | 定義 | 目標值 |
|:-----|:-----|:-------|
| **完成率** | 成功部署連結的學員 / 總學員 | ≥ 70% |
| **帶走率** | 作品集有連結 + 已加社群的學員 / 總學員 | ≥ 85% |

| **NPS** | 推薦分數 9-10 分的比例 | ≥ 50% |
| **轉化率** | 購買進階課程 / 總學員 | ≥ 10% |

## I) 風險清單 Top 5 + 對應備援

| # | 風險 | 機率 | 衝擊 | 備援方案 |
|:--|:-----|:-----|:-----|:---------|
| 1 | **環境地獄**（安裝失敗、版本衝突） | 高 | 高 | 課前強制完成安裝 + 備用 Codespaces 連結 + solution.py |
| 2 | **程度落差**（有人秒殺、有人掉隊） | 高 | 中 | 快的人給挑戰題（加 Dark Mode）、慢的人直接發解答 |
| 3 | **時間不夠**（Debug 吃掉時間） | 中 | 高 | 每輪預留 10min Buffer + 5 分鐘規則 + 砍 Reflection 時間 |
| 4 | **網路斷線**（雲端服務失效） | 低 | 高 | 手機熱點 × 2 + 離線版 Skeleton Code + 本地 Demo 替代 |
| 5 | **API 掛掉**（第三方服務異常） | 低 | 中 | 預存 Mock 資料 + 改用備用 API |

## K) 開課檢查清單總表

### MVP Checklist（最小可行版本）

| 檢查項目 | 具體內容 | 目的 |
|:---------|:---------|:-----|
| **Notion Hub** | 至少 5 頁：總覽/安裝/Prompt包/作品牆/問卷 | 建立學習資源中心，提供統一資訊入口 |
| **Skeleton Code** | exercise + solution 分支 | 提供練習框架與標準解答，確保學習進度 |
| **簽到表 + 學員名牌** | 報名學員清單與個人識別 | 掌握出席狀況，促進互動交流 |
| **備用熱點** | 至少 1 個行動網路熱點 | 預防網路斷線風險，確保課程順利進行 |
| **助教配置** | 至少 1 名（熟悉 solution.py） | 即時支援學員，處理個別技術問題 |
