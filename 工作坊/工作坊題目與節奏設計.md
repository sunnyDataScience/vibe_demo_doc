# AI Vibe Coding 工作坊：題目與節奏設計

> 解決你卡住的兩件事：**題目怎麼出** + **練習節奏怎麼控**

---

## 核心原則（先內化再往下看）

```
1. 題目不是考試，是「讓學員成功一次」的腳手架
2. 每道題都要有「完成條件 DoD」+「卡住退路 Fallback」
3. 節奏 = Context(15) → Demo(15) → Hands-on(45) → Reflection(10) → 休息(10)
4. 你追的是「跑通」，不是「完美」
```

---

## Part 1：題目設計框架

### 題目三層結構（每個模組都套用）

| 層級 | 目的 | 成功率 | 時間 | 設計原則 |
|:-----|:-----|:-------|:-----|:---------|
| **暖身題** | 先成功一次，建立信心 | 95% | 15-20min | 只有 1-2 個步驟，必定跑通 |
| **核心題** | 真正有價值，可放作品集 | 70% | 30-40min | 3-5 個步驟，有挑戰但可救 |
| **挑戰題** | 給高手，不拖垮全班 | 自選 | 剩餘時間 | 選做，不講解，只給提示 |

### 題目設計公式

```
【題目名稱】一句話說清楚要做什麼
【情境】為什麼要做這件事（痛點）
【輸入】學員拿到什麼（檔案/連結/骨架）
【產出】做完要看到什麼（截圖/連結/數字）
【DoD】怎麼算完成（可檢查的條件）
【Fallback】卡住怎麼辦（checkpoint/solution）
【Prompt 提示】給 AI 的指令參考（讓學員抄）
```

#### M0 暖身題：痛點盤點（20min）

```

  □ 寫出理想狀態（省多少時間 or 達成什麼效果）
  □ 能用一句話說給旁邊的人聽
【Fallback】講師給 3 個範例讓他選一個改
```

**範例痛點（給卡住的人選）**
1. 每月整理信用卡帳單，要開 5 個 App 複製貼上

```

【情境】

#### M4 暖身題：API 是什麼（15min）

```
【題目】用 Python 呼叫一次 API
【情境】理解「API = 自動幫你查資料的機器人」
【輸入】範例程式碼（5 行）
【產出】終端機印出今天的美金匯率
【DoD】

【範例程式碼】
```python

url = "https://api.exchangerate-api.com/v4/latest/USD"
response = requests.get(url)
data = response.json()
print(f"美金對台幣：{data['rates']['TWD']}")
```
```

## Part 3：節奏控制模板

### 單輪節奏（90-120 分鐘）

```
┌─────────────────────────────────────────────────────────┐
│  0:00  Context（15min）                                  │
│        - 為什麼做這件事                                   │
│        - 做完會長什麼樣（秀成果截圖）                      │
│        - 今天的 DoD 是什麼                               │
├─────────────────────────────────────────────────────────┤
│  0:15  Demo（15min）                                     │
│        - 講師完整跑一次（不跳步驟）                        │
│        - 標註 3 個「這裡會卡」的點                        │
│        - 結束前問：「有沒有問題？」                        │
├─────────────────────────────────────────────────────────┤
│  0:30  Hands-on（45min）                                 │
│        - 學員開始做                                       │
│        - 講師只說「下一步」（不重複講）                    │
│        - 助教巡場（5 分鐘修不好 → 發 checkpoint）         │
│        ┌─────────────────────────────────────┐          │
│        │ 0:45 倒數提醒：「剩 15 分鐘」        │          │
│        │ 0:55 倒數提醒：「剩 5 分鐘」         │          │
│        │ 1:00 時間到：「大家停一下」          │          │
│        └─────────────────────────────────────┘          │
├─────────────────────────────────────────────────────────┤
│  1:15  Reflection（10min）                               │
│        - 「誰成功了？舉手」                               │
│        - 邀請 1-2 人分享「你怎麼做到的」                  │
│        - 講師收斂：「這輪的關鍵是...」                    │
├─────────────────────────────────────────────────────────┤
│  1:25  休息（10min）                                     │
│        - 真的休息，不要偷講                               │
└─────────────────────────────────────────────────────────┘
```

### 講師話術模板

| 時機 | 話術 |
|:-----|:-----|
| Context 開場 | 「這輪我們要解決一個問題：______。做完之後，你會有 ______。」 |
| Demo 前 | 「我現在完整跑一次，你們先看，等等再動手。」 |
| Demo 中 | 「這裡很容易卡，等等如果你也遇到，就 ______。」 |
| Hands-on 開始 | 「現在開始，有問題舉手，助教會來。」 |
| 倒數提醒 | 「剩 15 分鐘，還沒跑通的人，可以先跳到 checkpoint。」 |
| Reflection | 「誰成功了？你是怎麼做到的？」 |
| 收斂 | 「這輪的關鍵是：______。回家你只要記得這一點。」 |

### 助教巡場 SOP

```
學員舉手
    │
    ├─ 看一眼錯誤訊息
    │      │
    │      ├─ 在 Top10 清單裡？ → 直接給解法
    │      │
    │      └─ 不在？ → 嘗試 Debug
    │              │
    │              ├─ 2 分鐘內解決 → OK
    │              │
    │              └─ 超過 2 分鐘 → 「我先給你 checkpoint，你先跟上，等等我回來」
    │
    └─ 完全不知道怎麼辦 → 叫講師暫停（可能是共通問題）
```

## Part 6：常見卡點 Top 10（助教背熟）

| # | 錯誤訊息 | 原因 | 解法 |
|:--|:---------|:-----|:-----|
| 1 | `ModuleNotFoundError` | 沒裝套件 | `pip install xxx` |
| 2 | `FileNotFoundError` | 路徑錯 | 檢查檔案位置，用絕對路徑 |
| 3 | `IndentationError` | 縮排錯 | 檢查空格/Tab 混用 |
| 4 | `SyntaxError` | 少括號/冒號 | 看錯誤行數，補上符號 |
| 5 | `KeyError` | 欄位名稱錯 | `df.columns` 檢查欄位名 |
| 6 | `JSONDecodeError` | API 回傳格式錯 | 先 `print(response.text)` |
| 7 | `ConnectionError` | 網路問題 | 檢查 WiFi，或用 mock 資料 |
| 8 | `PermissionError` | 檔案被鎖 | 關掉 Excel，重跑 |
| 9 | Streamlit 白屏 | 程式碼有錯 | 看終端機的紅字 |
| 10 | 部署失敗 | requirements 錯 | 用 `pipreqs` 重新生成 |
