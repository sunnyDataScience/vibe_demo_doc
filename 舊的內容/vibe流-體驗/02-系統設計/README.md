## 🧠 設計思維
### 從外到內
1. **用戶體驗** → 界面設計
2. **業務流程** → 系統邏輯
3. **技術實現** → 程式架構

### 從大到小
1. **整體架構** → 系統全貌
2. **模組分工** → 各部分職責
3. **實作細節** → 程式碼層級

## 💡 設計原則（用日常比喻）

### 1. 單一職責原則
就像餐廳分工：
- 廚師專心煮菜
- 服務生專心送餐
- 收銀員專心結帳

**應用**：每個程式模組只做一件事

### 2. 開閉原則
就像手機充電器：
- 可以換不同線頭（擴展）
- 但充電器本體不用改（封閉）

**應用**：容易新增功能，但不破壞原有程式

### 3. 依賴反轉原則
就像插座設計：
- 電器不需要知道電廠在哪
- 只要插座介面一樣就能用

**應用**：模組之間透過介面溝通，不直接依賴

## ⏰ 預估時間
- 理解概念：45分鐘
- 完成設計：2小時
- 總計：2.5小時

## 🎓 學習重點
1. **系統思維**：看懂複雜系統的結構
2. **抽象思考**：把具體問題變成通用模式
3. **溝通技巧**：用圖表清楚表達想法
4. **設計權衡**：在簡單和功能之間找平衡
