# 📋 階段一：需求分析與規劃 (PRD)

## 🎯 這個階段要做什麼？
把你腦海中的想法，變成清楚的計畫書。就像蓋房子前要先有建築圖一樣。

## 📝 產出文件
- `產品需求文件.md` - 我們要建立什麼產品
- `用戶故事地圖.md` - 用戶會怎麼使用
- `功能清單.md` - 詳細功能列表
- `問題拆解工作坊.md` - 教你怎麼問對問題

## 🚀 開始之前
想像你是一個忙碌的上班族，每天都想要快速了解重要新聞，但又沒有時間一篇篇慢慢看。你希望有個助手能：
- 幫你看完所有重要新聞
- 用你喜歡的方式總結給你聽
- 定期自動更新
- 隨時都能查看

這就是我們要建立的產品！

## ⏰ 預估時間
- 閱讀理解：30分鐘
- 完成練習：1小時
- 總計：1.5小時

## 🎓 學習重點
1. **需求分析**：如何把模糊的想法變成具體的需求
2. **用戶思維**：站在使用者角度思考問題
3. **優先順序**：分辨哪些功能最重要
4. **可行性評估**：判斷哪些現在能做，哪些以後再說
