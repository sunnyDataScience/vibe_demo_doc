# 🎯 Vibe Coding 工作坊課程統整文件

## 📋 文件總覽

本課程設計旨在讓學員**不寫程式碼,但建構系統思維**,透過積木式的模組化開發方式,學習如何與AI協作建構完整的訂單管理系統。

---

## 🎓 課程核心理念

```
┌─────────────────────────────────────────────────────────┐
│  「不寫程式碼,但建構系統思維」                         │
│   從需求 → 架構 → 流程 → AI實現 → 文件 → 整合          │
└─────────────────────────────────────────────────────────┘
```

**核心價值:**
- **思維優先**:先懂概念,再用AI實現
- **模組化設計**:複雜系統拆解成獨立積木
- **文件驅動**:記錄設計思路,確保可維護性
- **迭代開發**:先跑起來,再逐步完善

---

## 📚 課程結構:三階段學習法

### 【第一階段】俯瞰全局(2小時)

**目標**:建立完整系統的心智模型

**核心活動:**
1. **用戶故事地圖(30分鐘)**
   - 透過角色扮演理解不同使用者需求
   - 產出:`01_使用者旅程地圖.md`

2. **積木拆解(30分鐘)**
   - 用樂高比喻理解模組化
   - 識別6個核心模組及依賴關係
   - 產出:`02_模組依賴關係圖.md`

3. **資料流動模擬(60分鐘)**
   - 角色扮演:客戶下單完整流程
   - 理解資料在模組間的流動
   - 產出:`03_資料流程圖.md`

**關鍵概念:**
```
完整系統架構:
前端儀表板(Lovable)
    ↓
會員登入 + 後台管理
    ↓
雲端資料庫(Supabase)
    ↓
Google Sheet + CRM系統
```

---

### 【第二階段】模組開發(12小時)

**開發原則**:SDD (Structure-Driven Development)

每個模組遵循統一開發流程:
1. 第一性原理拆解(20分鐘)
2. 架構設計(30分鐘)
3. Vibe Prompt設計(20分鐘)
4. AI協作開發(40分鐘)
5. 文件產出(30分鐘)

**6個核心模組:**

#### 模組1:會員登入系統(2.5小時)
- **核心概念**:身份驗證(Authentication)
- **關鍵問題**:「怎麼證明你是你?」
- **技術要點**:註冊、登入、登出、權限判定
- **文件位置**:`docs/模組1_會員登入系統.md`

#### 模組2:後台管理系統(2.5小時)
- **核心概念**:權限管理(Authorization)
- **關鍵問題**:「不同的人能做不同的事」
- **技術要點**:角色定義、權限分配、中介層設計
- **文件位置**:`docs/模組2_後台管理系統.md`

#### 模組3:前端儀表板(2小時)
- **核心概念**:使用者介面設計(UI/UX)
- **關鍵問題**:「怎麼讓不懂電腦的人也能輕鬆使用?」
- **技術要點**:資訊架構、互動設計、元件化思維
- **文件位置**:`docs/模組3_前端儀表板.md`

#### 模組4:雲端資料庫(2小時)
- **核心概念**:資料建模(Data Modeling)
- **關鍵問題**:「現實世界的東西,怎麼存進電腦?」
- **技術要點**:實體識別、屬性定義、關係建立、資料完整性
- **文件位置**:`docs/模組4_雲端資料庫.md`

#### 模組5:Google Sheet整合(2小時)
- **核心概念**:API整合(API Integration)
- **關鍵問題**:「怎麼讓兩個不同的系統互相傳資料?」
- **技術要點**:認證授權、資料轉換、容錯機制
- **文件位置**:`docs/模組5_Google_Sheet整合.md`

#### 模組6:CRM客戶管理系統(2.5小時)
- **核心概念**:業務邏輯設計(Business Logic)
- **關鍵問題**:「怎麼追蹤客戶的狀態,提供更好的服務?」
- **技術要點**:客戶分類、互動記錄、狀態追蹤、規則引擎
- **文件位置**:`docs/模組6_CRM客戶管理系統.md`

---

### 【第三階段】系統整合(2小時)

**目標**:積木組裝,跑通完整流程

#### 活動1:積木組裝(60分鐘)
**完整流程串接:**
```
客戶下單 → 訂單處理 → CRM更新 → 週報產生

檢核清單:
✓ 各模組獨立運作正常
✓ 前端與後端串接
✓ 訂單與CRM串接
✓ 定時任務設定
```

**產出**:`整合測試報告.md`

#### 活動2:壓力測試(30分鐘)
測試異常情況:
- 重複下單
- 錯誤資料
- 權限越界
- API失效
- 大量資料

**產出**:`壓力測試報告.md`

#### 活動3:費曼學習法應用(30分鐘)
- 每位學員選一個模組
- 5分鐘簡報向他人解釋
- 回答提問,反思改進

**產出**:`模組X_費曼學習筆記.md`

## 🔑 核心教學原則

### 1. 第一性原理思維
每個模組都從最根本的問題出發:
- 會員登入:「怎麼證明你是你?」
- 權限管理:「不同的人能做不同的事」
- 資料庫:「現實世界的東西,怎麼存進電腦?」

### 2. 生活類比教學
用日常生活的例子解釋抽象概念:
- API = 郵局寄信
- 資料庫 = 圖書館
- 認證 = 門禁卡

### 3. 視覺化優先
- 用白板、便利貼、流程圖
- 減少文字說明
- 讓抽象概念具象化

### 4. 結構化Prompt設計
教學生如何寫好Prompt:
- ❌ 不好:「幫我做會員登入」
- ✅ 好:包含背景、功能需求、驗證規則、技術需求、介面需求

### 5. 可維護性思考
在開發過程中不斷引導:
- 模組化:「未來要改規則,需要改幾個地方?」
- 擴展性:「未來要加新功能,現在的架構好加嗎?」
- 可讀性:「6個月後別人看得懂你的設計嗎?」

## 💡 教學技巧建議

### 多用類比
「資料庫就像圖書館,API就像郵局,認證就像門禁卡」

### 視覺化優先
用白板、便利貼、流程圖,減少文字說明

### 失敗也是學習
「AI生成的程式碼有bug?很好!我們來看錯誤訊息,告訴AI怎麼修正」

### 小步快跑
每個模組先做最簡單的版本能跑,再慢慢加功能

### 鼓勵提問
「沒有笨問題,每個問題都幫助我們把系統想得更清楚」

## 🎓 結語

這套課程的核心價值在於:

**「不寫程式碼,但建構系統思維」**

學員離開教室時,帶走的不是一堆看不懂的code,而是:
- ✨ 能用自然語言指揮AI建構完整系統的能力
- 🧠 運算思維與系統設計能力
- 📚 完整的專案文件與可維護的架構
- 🚀 持續學習與擴展系統的信心

**記住:系統的價值不在於程式碼有多複雜,而在於它能解決真實的問題,並且能被持續維護與改進。**
